# MediaKids Website Update Reports

This folder is for optional boss-ready reports after public website changes.

Use `.agent/workflows/website-update-report.md` before creating a report.

Reports are for website-facing updates such as:

- New images, albums, or galleries
- New sections or UI changes
- Video/popup updates
- Security, SEO, performance, accessibility, or broken-link improvements

Do not create a boss-ready report for internal setup work such as skills, SSH, workflow docs, or handbooks unless the owner asks.
